# 条件式多周期共振策略研究报告

## 1. 研究目标

本研究围绕一个以“多周期共振”为核心的条件式策略展开。目标是在 `000852.XSHG` 上，使用项目当前框架中的原生信号，构建一个可以通过 `Position/Event` 体系表达并回测的策略，重点考察：

- 策略是否能够使用 `Signal -> Event -> Position -> CzscStrategyBase.backtest` 的项目推荐路径实现；
- 策略在 `000852.XSHG` 上是否满足年化收益 20% 以上、最大回撤 10% 以下；
- 策略迁移到 `000905.XSHG` 和 `159915.XSHE` 后是否仍具备稳健性；
- 后续应如何优化，而不是仅针对单一标的过拟合。

## 2. 策略逻辑

策略定位为“**大周期定方向，小周期找进出场**”的条件式多周期共振策略，不是多信号打分模型。

使用信号如下：

| 角色 | 周期 | 信号 |
|---|---|---|
| 大周期方向过滤 | 60分钟 | `60分钟_D1SMA#5_分类V221101` |
| 小周期入场确认 | 15分钟 | `15分钟_D1N64T10_绝对动量V230227` |
| 小周期出场确认 | 15分钟 | `15分钟_D1N16T10_绝对动量V230227` |

具体交易规则：

| 动作 | 条件 |
|---|---|
| 开多 | 60分钟 SMA#5 分类为 `多头`，且 15分钟 N64 绝对动量为 `超强` |
| 平多 | 15分钟 N16 绝对动量为 `超弱`，或 60分钟 SMA#5 分类转为 `空头` |
| 开空 | 60分钟 SMA#5 分类为 `空头`，且 15分钟 N64 绝对动量为 `超弱` |
| 平空 | 15分钟 N16 绝对动量为 `超强`，或 60分钟 SMA#5 分类转为 `多头` |

交易含义：

- 60分钟信号决定交易方向，只在大周期多头环境中寻找多头机会，只在大周期空头环境中寻找空头机会。
- 15分钟 N64 动量用于入场，要求动量足够极端，避免普通波动频繁触发。
- 15分钟 N16 动量用于出场，窗口更短，对反向动量更敏感，用来快速退出。
- 大周期方向反转也是强制退出条件，避免小周期信号滞后导致持仓继续暴露在反向趋势中。

## 3. Position/Event 实现方式

策略已经使用项目原生 `Position/Event` 体系实现，脚本位置：

- `examples/signals_dev/backtest_conditional_resonance_position_event_000852.py`

实现上没有把多空逻辑塞进单个 `Position`，而是拆成两个独立持仓：

| Position | 职责 |
|---|---|
| `CR_Long_60mFilter_15mN64Entry_15mN16Exit` | 只处理开多和平多 |
| `CR_Short_60mFilter_15mN64Entry_15mN16Exit` | 只处理开空和平空 |

拆分原因：

当前 `Position` 引擎会按 `opens + exits` 的固定顺序寻找第一个匹配事件。若把多空逻辑放在同一个 `Position`，方向无关的平多事件可能在空头持仓时先匹配，从而挡住后续平空事件。拆成多头 / 空头两个 `Position` 后，每个 `Position` 只处理自己的方向，事件优先级与原条件式状态机一致。

最终组合权重计算方式：

```text
组合权重 = CR_Long.holds.pos + CR_Short.holds.pos
```

验证结果显示，`Position/Event` 版本与原显式状态机逐 15 分钟 K 线完全一致：

| 标的 | 对比K线数 | 差异K线数 | 权重绝对差合计 |
|---|---:|---:|---:|
| `000852.XSHG` | 24688 | 0 | 0.0 |
| `000905.XSHG` | 24688 | 0 | 0.0 |
| `159915.XSHE` | 24688 | 0 | 0.0 |

## 4. 回测方式

回测设置：

| 项目 | 设置 |
|---|---|
| 数据周期 | 15分钟基础 K 线，60分钟信号由框架合成 |
| 数据预热 | 从 2018-01-01 开始取数，用于信号预热 |
| 统计区间 | 2020-01-02 至 2026-05-20 |
| 手续费 | 单边 2BP，即 `fee_rate = 0.0002` |
| 回测路径 | `CzscStrategyBase.backtest` 生成 `signals / holds / pairs` |
| 绩效计算 | 基于 `holds.pos` 汇总为权重序列，按 15分钟收益扣费后聚合为日收益 |

主要输出目录：

| 标的 | 输出目录 |
|---|---|
| `000852.XSHG` | `examples/results/conditional_resonance_position_event_000852/` |
| `000905.XSHG` | `examples/results/conditional_resonance_position_event_000905_XSHG/` |
| `159915.XSHE` | `examples/results/conditional_resonance_position_event_159915_XSHE/` |

每个目录中包含：

- `summary_position_event.csv`：整体绩效；
- `period_stats_position_event.csv`：分阶段绩效；
- `weights_position_event.csv`：组合权重序列；
- `holds_position_event.csv`：两个 Position 的原始持仓；
- `pairs_position_event.csv`：交易对；
- `signals_position_event.csv`：信号明细；
- `positions/`：可复用的 Position JSON。

## 5. 回测结果

### 5.1 整体绩效

| 标的 | 年化收益 | 最大回撤 | 期末净值 | Calmar | 日胜率 | 换手次数 | 持仓占比 | 是否满足 20% / -10% |
|---|---:|---:|---:|---:|---:|---:|---:|---|
| `000852.XSHG` | 40.89% | -9.93% | 8.9073 | 4.12 | 47.57% | 3749 | 64.92% | 是 |
| `000905.XSHG` | 19.98% | -17.31% | 3.1969 | 1.15 | 44.72% | 3918 | 64.77% | 否 |
| `159915.XSHE` | 20.99% | -25.74% | 3.3719 | 0.82 | 46.27% | 3966 | 64.04% | 否 |

结论：

- `000852.XSHG` 达到原始目标，年化收益超过 20%，最大回撤小于 10%。
- `000905.XSHG` 年化收益接近 20%，但最大回撤明显超出 10%。
- `159915.XSHE` 年化收益略高于 20%，但最大回撤达到 -25.74%，风险暴露过大。
- 当前参数不能作为三标的统一策略直接使用。

- `159915.XSHE` 最近两年收益较高，但回撤也集中放大，说明趋势反转或高波动阶段对该标的冲击更强。

## 6. 策略评价

### 优点

- 逻辑清晰：大周期过滤方向，小周期负责触发和退出。
- 可以完全使用项目推荐的 `Position/Event` 体系实现，符合当前项目回测规范。
- `000852.XSHG` 上收益 / 回撤结构较好，且分阶段表现稳定。
- 多空拆分后，回测结果与原条件状态机逐 K 线完全一致，工程实现可靠。

### 问题

- 跨标的稳健性不足：`000905.XSHG` 与 `159915.XSHE` 均未满足最大回撤小于 10% 的要求。
- 策略持仓占比约 64%，暴露时间较长，遇到标的自身高波动阶段时回撤容易放大。
- 当前仅使用趋势方向和动量强弱，没有显式波动率过滤、止损、仓位缩放或风险预算机制。
- 当前参数是在 `000852.XSHG` 上表现较优，存在单标的适配风险，后续不能只继续围绕 `000852.XSHG` 做收益最大化。

## 7. 策略优化方向

### 7.1 先做稳健性优化，而不是单标的收益优化

当前 `000852.XSHG` 已经达到目标，下一步优化重点不应继续追求更高收益，而应降低 `000905.XSHG` 和 `159915.XSHE` 的回撤。建议将优化目标改为多标的联合目标：

```text
目标函数 = 平均年化收益 / 平均最大回撤惩罚 / 单标的最差回撤惩罚
```

筛选条件建议：

- 三个标的平均年化收益不低于 20%；
- 任一标的最大回撤不超过 15%，再逐步压到 10%；
- 参数不能只在 `000852.XSHG` 上最优；
- 分阶段表现不能依赖单一年份贡献。

### 7.2 参数搜索方向

可以在保持策略结构不变的前提下搜索参数：

| 模块 | 当前参数 | 可搜索范围 |
|---|---|---|
| 大周期过滤 | 60分钟 SMA#5 | 60分钟 SMA#5 / SMA#10 / SMA#20，或日线 SMA 过滤 |
| 入场动量窗口 | 15分钟 N64 | N48 / N64 / N96 / N128 |
| 入场阈值 | T10 | T10 / T15 / T20 |
| 出场动量窗口 | 15分钟 N16 | N8 / N16 / N24 / N32 |
| 出场阈值 | T10 | T5 / T10 / T15 |

优化时应使用 `Position/Event` 生成候选策略，不建议回退到自定义状态机。

### 7.3 增加波动率与风险过滤

`000905.XSHG` 和 `159915.XSHE` 的问题主要是回撤控制不足。可以考虑增加以下条件：

- 高波动阶段降低仓位或禁止开仓；
- 大幅跳空或连续亏损后进入冷却期；
- 使用 `Position.stop_loss` 引入合理 BP 止损，而不是设置为极大值；
- 使用 `Position.timeout` 限制过长持仓；
- 对 ETF 类标的单独评估是否禁用做空，避免不可交易方向造成理论回测偏差。

### 7.4 优化出场规则

当前出场规则偏向“反向动量出现才退出”。对高波动标的，可以增加更早的保护性退出：

- 多头入场后，若 60分钟趋势从 `多头_向上` 转为 `多头_向下`，提前减仓或退出；
- 空头入场后，若 60分钟趋势从 `空头_向下` 转为 `空头_向上`，提前减仓或退出；
- 引入利润回撤保护，例如盈利后回撤达到一定 BP 则平仓；
- 对 N16 退出信号做确认，减少刚入场后被噪声洗出的情况。

### 7.5 分标的参数或分组参数

如果目标是单一策略覆盖指数与 ETF，建议不要强行使用完全相同参数。可以考虑分组：

- 中证 1000 / 中证 500 指数类：使用同一组指数参数；
- 创业板 ETF：单独使用 ETF 参数，并加入更严格的回撤控制；
- 若要落地交易，还需要结合可交易性、融券 / 股指期货替代、滑点和成交量约束重新评估。

## 8. 结论

当前策略已经完成从研究型条件状态机到项目原生 `Position/Event` 架构的迁移，并且在三个标的上均验证了逐 K 线权重完全一致。

核心结论：

- `000852.XSHG`：策略达标，年化 40.89%，最大回撤 -9.93%，是当前版本的主要有效样本。
- `000905.XSHG`：收益接近目标，但最大回撤 -17.31%，需要加强风控。
- `159915.XSHE`：年化超过 20%，但最大回撤 -25.74%，当前参数不适合直接使用。

因此，当前版本可以作为 `000852.XSHG` 上的有效策略原型，但不能直接声明为跨标的通用策略。下一阶段研究重点应放在多标的联合稳健性、波动率过滤、止损 / 超时风控和分标的参数适配上。
