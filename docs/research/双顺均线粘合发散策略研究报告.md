# 双顺均线粘合发散策略研究报告

日期：2026-05-24

## 1. 研究目标

本研究围绕用户提供的“均线粘合发散”通达信指标和《双顺经验贴》中的策略思想，在当前 CZSC 项目框架下完成信号实现、策略组合、参数对比和多标的验证。

研究目标包括：

- 将通达信“均线联合发散”逻辑按用户命名要求实现为“均线粘合发散”信号；
- 在项目当前 Rust 信号体系中补充双顺相关信号，避免 Python 侧适配层；
- 搭建 60分钟一笔版本和 15分钟一笔版本；
- 对三种方向过滤方式做完整 `Position/Event` 精确回测；
- 在 `000852.XSHG`、`000905.XSHG`、`159915.XSHE` 上比较策略适配性；
- 形成后续优化建议。

本报告中的最终绩效结论均以完整 `Position/Event` 回测口径为准；参数初筛阶段的轻量状态机结果仅用于快速排序和候选筛选。

## 2. 信号实现

### 2.1 均线粘合发散信号

用户提供的原始通达信代码如下：

```text
MA5:=MA(C,5);
MA10:=MA(C,10);
MA20:=MA(C,20);
MA40:=MA(C,40);
MA60:=MA(C,60);
MA120:=MA(C,120);
MA250:=MA(C,250);
HH:=MAX(MAX(MAX(MA5,MA10),MAX(MA20,MA40)),MAX(MAX(MA60,MA120),MA250));
LL:=MIN(MIN(MIN(MA5,MA10),MIN(MA20,MA40)),MIN(MIN(MA60,MA120),MA250));
A1:=C>HH AND MA60>MA120;
A2:=MA60<MA120 OR C<LL;
B1:=C<LL AND MA60<MA120;
B2:=MA60>MA120 OR C>HH;
```

实现后的信号函数：

| 项目 | 内容 |
|---|---|
| Rust 函数 | `tas_ma_joint_V260520` |
| 文件 | `crates/czsc-signals/src/tas.rs` |
| 信号模板 | `{freq}_D{di}SMA{ma_seq}_均线粘合发散V260520` |
| 默认均线组 | `5#10#20#40#60#120#250` |
| 输出 | `看多` / `看空` / `其他` |

核心逻辑：

- 多头状态：价格上穿所有均线，且 MA60 > MA120 后进入 `看多`；
- 多头结束：MA60 < MA120 或价格跌破所有均线；
- 空头状态：价格跌破所有均线，且 MA60 < MA120 后进入 `看空`；
- 空头结束：MA60 > MA120 或价格上穿所有均线；
- 使用 `BARSLAST(A1) < BARSLAST(A2)` 的状态保持思想，而不是只看单根 K 线触发。

### 2.2 双顺中线突破信号

- 平多遇到结构 `看空`；
- 平空遇到结构 `看多`。

| 输出 v3 | `向上` / `向下` |

策略中使用方式：

- 开多要求低一级周期 MA60 支撑信号为 `看多`；

- 平多遇到低一级周期 `看空` 或 `其他_跌破`；
- 平空遇到低一级周期 `看多` 或 `其他_上破`。

## 3. 策略结构

### 3.1 60分钟一笔版本

60分钟一笔版本的周期搭配如下：

| 角色 | 周期 | 信号 |
|---|---|---|
| 方向过滤 | 日线 | `日线_D1SMA5#10#20#40#60#120#250_均线粘合发散V260520` |
| 结构触发 | 60分钟 | `60分钟_D1N3M3_双顺中线突破V260521` |
| 入场过滤 | 15分钟 | `15分钟_D1SMA#60T50_双顺均线支撑V260521` |
| 执行确认 | 5分钟 | `5分钟_D1SMA5#10#20#40#60#120#250_均线粘合发散V260520` |
| 执行确认 | 5分钟 | `5分钟_N20通道_突破信号V240623` |

交易规则：

| 动作 | 条件 |
|---|---|
| 开多 | 15分钟 MA60 支撑 `看多`，5分钟均线粘合发散 `看多`，5分钟 N20 通道突破 `做多`，60分钟中线突破为 `上破中线` 或 `向上确认`，再叠加方向过滤版本 |
| 平多 | 60分钟结构 `看空`，或 15分钟 MA60 `看空/跌破`，或 5分钟均线粘合发散 `看空` |
| 开空 | 15分钟 MA60 支撑 `看空`，5分钟均线粘合发散 `看空`，5分钟 N20 通道突破 `做空`，60分钟中线突破为 `下破中线` 或 `向下确认`，再叠加方向过滤版本 |
| 平空 | 60分钟结构 `看多`，或 15分钟 MA60 `看多/上破`，或 5分钟均线粘合发散 `看多` |

### 3.2 15分钟一笔版本

15分钟一笔版本是将 60分钟一笔版本整体下沉一级：

| 角色 | 周期 | 信号 |
|---|---|---|
| 方向过滤 | 60分钟 | `60分钟_D1SMA5#10#20#40#60#120#250_均线粘合发散V260520` |
| 结构触发 | 15分钟 | `15分钟_D1N3M3_双顺中线突破V260521` |
| 入场过滤 | 5分钟 | `5分钟_D1SMA#60T50_双顺均线支撑V260521` |
| 执行确认 | 1分钟 | `1分钟_D1SMA5#10#20#40#60#120#250_均线粘合发散V260520` |
| 执行确认 | 1分钟 | `1分钟_N20通道_突破信号V240623` |

交易规则与 60分钟版本一致，只是所有周期下沉一级。

## 4. 三种方向过滤版本

本研究主要比较三种方向过滤方式。三者除方向过滤外，其余结构、支撑、执行、平仓规则都一致。

| 版本 | 参数 | 含义 | 开多方向条件 | 开空方向条件 |
|---|---|---|---|---|
| `direction_none` | `direction_mode="none"` | 不看更大周期方向 | 无 | 无 |
| `base` | `direction_mode="not_opposite"` | 更大周期不能反向 | 排除 `看空` | 排除 `看多` |
| `direction_same` | `direction_mode="same"` | 更大周期必须同向 | 必须 `看多` | 必须 `看空` |

直观理解：

```text
direction_none = 不过滤
base           = 不逆大势
direction_same = 顺大势才做
```

研究过程中发现，`base` 并不一定是最稳妥的折中方案。它允许更大周期处于 `其他/中性` 状态时开仓，这在部分标的上会增加趋势切换区间的暴露，导致回撤放大。

## 7. 000852 研究过程与结果

### 7.1 15分钟一笔参数初筛

在完整 `Position/Event` 验证前，先用 `union_signals + 轻量状态机` 做过单因子参数初筛。该阶段用于快速比较方向过滤、结构窗口、支撑阈值、突破通道长度。

### 9.2 方向过滤不是统一答案

研究最重要的结论是：方向过滤方式不能简单统一。

| 标的 | 适配特征 |
|---|---|
| `000852.XSHG` | 趋势弹性强，取消大周期方向过滤后能捕捉更多有效波段，15分钟 `direction_none` 表现最强 |
| `000905.XSHG` | 偏中间状态，15分钟一笔中 `base` 更均衡，60分钟一笔中 `direction_none` 略优 |
| `159915.XSHE` | 对短周期噪声更敏感，15分钟一笔必须同向过滤；60分钟一笔收益高但回撤大 |

这说明“双顺”逻辑对不同资产的波动结构高度敏感。直接用一组参数覆盖所有标的并不稳妥。

### 9.3 15分钟与 60分钟的差异

| 维度 | 15分钟一笔 | 60分钟一笔 |
|---|---|---|
| 交易频率 | 高 | 中 |
| 单笔收益 | 较低 | 较高 |
| 对滑点敏感度 | 高 | 中 |
| 对方向过滤依赖 | 更强，尤其 ETF | 相对较弱 |
| 适合用途 | 捕捉更细波段，适合高流动性和低成本环境 | 捕捉较大波段，更易实盘消化 |

`000852.XSHG` 上 15分钟一笔明显优于 60分钟一笔；但在 `000905.XSHG` 和 `159915.XSHE` 上，15分钟一笔并没有形成压倒性优势。特别是 `159915.XSHE`，60分钟 `direction_none` 收益最高，但回撤很大；15分钟 `direction_same` 更稳但收益下降。

## 10. 风险与局限

### 10.1 做空可交易性

本研究使用多空权重回测。对指数标的，这更接近使用股指期货或其他衍生品表达；对 ETF 标的，现货账户直接做空不可行，需要通过融券、期权、期货替代或只保留多头逻辑重新验证。

因此，`159915.XSHE` 的多空结果应视为理论策略表现，不应直接等同于 ETF 现货实盘。

### 10.2 交易成本

本研究固定手续费万2，未引入滑点和冲击成本。对 15分钟一笔版本尤其需要注意：

- 交易笔数高；
- 单笔收益 BP 明显低于 60分钟版本；
- 实盘滑点可能显著侵蚀收益。

例如 `000905.XSHG` 的 15分钟 `direction_none` 单笔均值只有 9.55bp，虽然年化较高，但对成交质量更敏感。

### 10.3 参数解释性

`support_t30 / support_t80` 初筛结果与 `support_t50` 完全一致，说明当前策略没有真正把 MA60 支撑阈值变成核心过滤条件。该参数目前不具备有效解释力，不应纳入正式优化结论。

### 10.4 回测区间集中

统计区间为 2020-2026，覆盖了疫情冲击、2021-2022 调整、2024-2025 大波动阶段，但仍属于单一历史样本。后续若要进入实盘前研究，应继续做：

- 分年度收益贡献拆解；
- 最大回撤区间复盘；
- 多标的滚动样本验证；
- 参数稳定性检验；
- 滑点敏感性分析。

## 11. 后续研究建议

### 11.1 对 000852 的建议

优先候选：

| 方案 | 理由 |
|---|---|
| 15分钟 `direction_none` | 收益与 Calmar 最强，是当前主策略候选 |
| 15分钟 `direction_same` | 回撤最低、单笔质量更高，可作为稳健低频版本 |

下一步建议：

- 复盘 15分钟 `direction_none` 的最大回撤区间；
- 检查收益是否集中在少数年份；
- 加入滑点假设后重新评估；
- 若要实盘化，先考虑减少交易次数或加入冷却期。

### 11.2 对 000905 的建议

优先候选：

| 方案 | 理由 |
|---|---|
| 15分钟 `base` | Calmar 与 60分钟 `direction_none` 接近，但回撤更低 |
| 60分钟 `direction_none` | 收益略高，交易次数低于 15分钟版本 |

下一步建议：

- 比较 15分钟 `base` 与 60分钟 `direction_none` 的年度收益稳定性；
- 尝试增加波动率过滤，压低 -11% 到 -13% 左右的回撤；
- 测试只做多版本，排除指数做空实现问题。

### 11.3 对 159915 的建议

优先候选：

| 方案 | 理由 |
|---|---|
| 15分钟 `direction_same` | 在 15分钟版本中明显最稳 |
| 60分钟 `direction_none` | 收益最高，但最大回撤过大，只适合继续研究，不宜直接采用 |

下一步建议：

- 重点研究只做多版本；
- 对 60分钟 `direction_none` 增加止损或波动率过滤；
- 对 15分钟 `direction_same` 尝试提高收益，例如放宽执行确认或优化出场。

### 11.4 统一策略与分标的策略

当前结果不支持“一套方向过滤参数覆盖所有标的”。更合理的路线是：

```text
先按标的类别分组，再做组内统一参数。
```

建议分组：

- 中证 1000 / 中证 500 指数类：可测试较松方向过滤；
- ETF 类：需要更严格方向过滤，并优先评估只做多；
- 若后续加入更多品种，可按波动率、趋势持续性、成交成本分层。

## 12. 结论

本研究完成了从信号实现到完整 `Position/Event` 精确验证的闭环。核心结论如下：

1. `均线粘合发散`、`双顺中线突破`、`双顺均线支撑` 三类信号已经可以在当前 Rust 信号框架下表达双顺策略。
2. 15分钟一笔和 60分钟一笔都能通过 `Signal -> Event -> Position` 体系完整实现。
3. `000852.XSHG` 是当前策略最适配标的，15分钟一笔 `direction_none` 表现显著最好，年化 45.53%，最大回撤 -10.48%，Calmar 4.35。
4. `000905.XSHG` 表现中等，15分钟 `base` 与 60分钟 `direction_none` 都可继续研究，但收益水平低于 000852。
5. `159915.XSHE` 对方向过滤更敏感，15分钟必须使用 `direction_same` 才能控制风险；60分钟 `direction_none` 虽有较高收益，但回撤过大。
6. `base` 即“不反向过滤”并不是稳定最优，它在多个场景中回撤偏大。
7. 后续重点不应继续单纯提高收益，而应围绕滑点、只做多、波动率过滤、止损、分标的参数稳定性展开。

当前可作为下一阶段精细化研究的候选组合：

| 标的 | 推荐继续研究版本 |
|---|---|
| `000852.XSHG` | 15分钟 `direction_none`，备选 15分钟 `direction_same` |
| `000905.XSHG` | 15分钟 `base`，备选 60分钟 `direction_none` |
| `159915.XSHE` | 15分钟 `direction_same`，谨慎观察 60分钟 `direction_none` |
