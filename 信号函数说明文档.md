# CZSC 信号函数说明文档

本文档适用于 CZSC 1.0 之后的 Rust 信号系统。1.0 开始，信号函数主体不再维护在 `czsc/signals/*.py`，而是位于：

```text
crates/czsc-signals/src/
```

Rust 侧通过 `#[signal(...)]` 宏声明信号元信息，并注册到统一信号表。Python 侧通过 `czsc._native.signals` 和 `czsc.traders` 调度这些信号。


## 信号分类

CZSC 1.0 的信号大致分为两类：

- **K-line 信号**：只依赖单个 `CZSC` 分析对象，可以通过 `czsc._native.signals.call_signal` 直接调用。
- **Trader 信号**：依赖 `CzscTrader`、多周期状态或持仓状态，应通过 `CzscSignals` / `CzscTrader` / `generate_czsc_signals` 调度。

## K-line 信号直接调用

```python
from czsc import CZSC
from czsc._native.signals import call_signal

c = CZSC(bars)
sig = call_signal("bar_r_breaker_V230326", c, {})[0]
print(sig.key)
print(sig.value)
print(sig.to_string())
```

带参数示例：

```python
sig = call_signal("zdy_macd_bc_V230422", c, {"di": 1, "th": 50})[0]
```

`call_signal` 返回 `list[Signal]`。常用属性：

- `sig.key`：信号键，通常由 `k1_k2_k3` 构成
- `sig.value`：信号值，通常由 `v1_v2_v3_score` 构成
- `sig.to_string()`：完整信号字符串

## 批量信号生成

研究和回测中更常用 `signals_config`：

```python
from czsc.traders import generate_czsc_signals

signals_config = [
    {"name": "cxt_bi_status_V230101", "freq": "30分钟"},
    {"name": "tas_ma_base_V221101", "freq": "日线", "di": 1, "timeperiod": 5, "ma_type": "SMA"},
    {"name": "bar_zdt_V230331", "freq": "30分钟", "di": 1},
]

df = generate_czsc_signals(
    bars,
    signals_config=signals_config,
    sdt="2024-01-01",
    df=True,
)
```

流式计算：

```python
from czsc import BarGenerator
from czsc.traders import CzscSignals

bg = BarGenerator(base_freq="30分钟", freqs=["30分钟", "日线"], max_count=5000)
cs = CzscSignals(bg, signals_config=signals_config)

for bar in bars:
    cs.update_signals(bar)

print(cs.s)
```

## 信号字符串

完整信号字符串格式：

```text
{k1}_{k2}_{k3}_{v1}_{v2}_{v3}_{score}
```

示例：

```text
30分钟_D1_表里关系V230101_向上_任意_任意_0
日线_D1SMA#5_分类V221101_多头_任意_任意_0
5分钟_RBreaker_BS辅助V230326_做多_趋势_任意_0
```

事件匹配时使用完整字符串：

```python
from czsc import Event

event = Event.load(
    {
        "name": "表里向上开多",
        "operate": "开多",
        "signals_all": ["30分钟_D1_表里关系V230101_向上_任意_任意_0"],
    }
)
```

## 新增信号开发规范

新增信号应遵循以下约束：

- 在 `crates/czsc-signals/src/` 中选择合适模块，或新增 `.rs` 模块后登记到 `lib.rs` / 相关 registry。
- 使用 `#[signal(category = "...", name = "...", template = "...", opcode = "...", param_kind = "...")]` 声明元信息。
- 参数解析逻辑放在 Rust 侧，避免 Python wrapper 做默认值补齐或类型归一化。
- K-line 信号应只依赖 `CZSC`、参数和 `TaCache`。
- Trader 信号依赖跨周期或持仓状态时，明确标记为 trader 类信号，并通过交易器路径调度。
- 增加 Rust 单测；Python 侧如涉及公共 API，也补充 `tests/` 下的回归测试。
- 如新增 `.rs` 文件，同步更新 `scripts/dump_signal_catalog.py` 的 `FILE_METADATA`，避免文档生成时落入“未分类”。

## 常见迁移

| 旧写法 | 1.0 后写法 |
|---|---|
| `from czsc.signals.cxt import cxt_bi_status_V230101` | `signals_config=[{"name": "cxt_bi_status_V230101", "freq": "..."}]` |
| 直接调用 Python 信号函数返回 `OrderedDict` | `call_signal(...)[0]` 返回 `Signal` 对象 |
| 在 `czsc/signals/*.py` 新增信号 | 在 `crates/czsc-signals/src/*.rs` 新增 Rust 信号 |
| `from czsc.traders.base import generate_czsc_signals` | `from czsc.traders import generate_czsc_signals` |
