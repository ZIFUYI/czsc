# 聚宽数据源配置说明

## 配置说明

本项目支持通过聚宽 JQData 获取行情数据。账号密码通过本地环境变量配置，不要提交到 Git。

## 📁 相关文件

- `.env` - 环境变量配置文件（包含账号密码）
- `config_jqdata.py` - 配置测试脚本
- `examples/jqdata_demo.py` - 使用示例
- `czsc/connectors/jq_connector.py` - 聚宽连接器（已修复兼容性问题）

## 账号信息

- 用户名：从 `JQDATA_USERNAME` 环境变量读取
- 密码：从 `JQDATA_PASSWORD` 环境变量读取
- 剩余查询次数：可通过 `get_query_count()` 查询

## 🚀 使用方法

### 1. 基础用法

```python
from czsc.connectors.jq_connector import set_token, get_kline
import os
from dotenv import load_dotenv

# 加载环境变量
load_dotenv()

# 设置Token（首次使用）
set_token(
    jq_mob=os.getenv('JQDATA_USERNAME'),
    jq_pwd=os.getenv('JQDATA_PASSWORD')
)

# 获取日线数据
bars = get_kline(
    symbol="000001.XSHE",  # 股票代码
    freq="D",              # 日线
    start_date="2024-01-01",
    end_date="2024-12-31",
)

# 获取分钟线数据
bars = get_kline(
    symbol="000001.XSHE",
    freq="5min",           # 5分钟线
    start_date="2024-12-01",
    end_date="2024-12-31",
)
```

### 2. 结合CZSC进行缠论分析

```python
from czsc import CZSC

# 获取数据
bars = get_kline(
    symbol="000001.XSHE",
    freq="D",
    start_date="2023-01-01",
    end_date="2024-12-31",
)

# 创建CZSC对象
czsc = CZSC(bars)

# 查看分析结果
print(f"分型数量: {len(czsc.fx_list)}")
print(f"笔数量: {len(czsc.bi_list)}")
print(f"最后一笔方向: {czsc.bi_list[-1].direction}")
```

### 3. 支持的周期

- `1min` - 1分钟
- `5min` - 5分钟
- `15min` - 15分钟
- `30min` - 30分钟
- `60min` - 60分钟
- `D` - 日线
- `W` - 周线
- `M` - 月线

### 4. 股票代码格式

- 上交所：`600000.XSHG`（如：浦发银行）
- 深交所：`000001.XSHE`（如：平安银行）
- 创业板：`300xxx.XSZE`
- 科创板：`688xxx.XSHG`

## 📝 其他功能

### 查询剩余次数

```python
from czsc.connectors.jq_connector import get_query_count

count = get_query_count()
print(f"剩余查询次数: {count}")
```

### 获取指数成分股

```python
from czsc.connectors.jq_connector import get_index_stocks

# 获取沪深300成分股
symbols = get_index_stocks("000300.XSHG", date="2024-12-31")
```

### 获取概念股

```python
from czsc.connectors.jq_connector import get_concept_stocks

# 获取某概念的成分股
symbols = get_concept_stocks("GN036", date="2024-12-31")
```

## ⚠️ 注意事项

1. `.env` 文件包含敏感信息，不要提交到 Git
2. 聚宽有查询频率限制，建议合理使用缓存
3. Token 会保存在 `~/jq.token` 文件中，设置一次即可
